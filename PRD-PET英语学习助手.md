# PET 英语学习助手 — 产品需求文档 (PRD)

**版本**: v1.0
**日期**: 2026-07-19
**目标用户**: 小学五年级学生（已通过 KET，正在备考 PET）

---

## 1. 项目背景

### 1.1 用户画像

- 小学五年级学生，约 10-11 岁
- 已通过 KET 考试（词汇量约 1500-1800 词）
- 正在备考 PET（B1 Preliminary，词汇量需提升至约 3500 词）
- 当前痛点：单词拼写不扎实，进而影响听力和阅读理解

### 1.2 问题分析

PET 相比 KET，词汇量要求增长近一倍。学生面临的典型困境是：

- **拼写薄弱**：能认识单词但拼写时容易出错（漏字母、字母顺序颠倒、相似词混淆）
- **听力受影响**：因拼写不熟练，听到单词后无法快速反应对应的拼写和含义
- **阅读受影响**：词汇量不足导致阅读速度慢、理解偏差

### 1.3 核心思路

本工具聚焦「**从认识 → 能拼写**」这一核心跨越。研究表明，主动回忆（active recall）和间隔重复（spaced repetition）是词汇学习最有效的手段。工具围绕「听音拼写」和「间隔复习」两个核心机制构建，帮助学生在拼写练习中同步强化听力和词汇记忆。

---

## 2. 产品定位

一款**简洁实用**的 PET 词汇学习网页工具，支持电脑和移动端访问。不做花哨的游戏化包装，而是通过高效的练习机制帮助孩子扎实掌握 PET 单词的拼写、发音和含义。

**核心差异化价值**：不是「背单词」，而是「练拼写」——把被动认识转化为主动输出。

---

## 3. 词汇内容规划

### 3.1 词汇来源

基于剑桥官方 **B1 Preliminary Vocabulary List（2025 年 8 月版）**，整理 PET 核心词汇约 2000-2500 个（剔除已掌握的 KET 基础词汇后），覆盖以下话题领域：

| 话题 | 示例内容 |
|------|---------|
| 服饰与配饰 | backpack, blouse, hoodie, trainers, bracelet |
| 通讯与科技 | app, blog, download, smartphone, social media, Wi-Fi |
| 教育 | curriculum, laboratory, qualification, certificate |
| 娱乐与媒体 | documentary, podcast, filmmaker, chat show |
| 食物与饮料 | barbecue, broccoli, coconut, burger, recipe |
| 健康与运动 | athletics, fitness, injury, ambulance, clinic |
| 旅行与交通 | boarding pass, cruise, destination, luggage, flight |
| 情感与性格 | ambitious, anxious, confident, curious, grateful |
| 职业与工作 | architect, candidate, colleague, employer, interview |
| 自然环境 | climate change, cliff, continent, atmosphere, wildlife |

### 3.2 词汇分级

将词汇按难度和使用频率分为三个等级：

- **L1 基础必备**（约 800 词）：PET 高频核心词，日常交流必备
- **L2 进阶提升**（约 800 词）：PET 常见话题词，阅读听力高频出现
- **L3 拓展挑战**（约 400 词）：较抽象或使用频率较低的词汇

### 3.3 单词条目结构

每个单词包含以下信息：

- 英文拼写
- 英式发音音标 + 美式发音音标
- 音频发音（使用浏览器 TTS 或预置音频）
- 中文释义（1-2 个核心义项）
- 词性
- 一个简单例句（带中文翻译）
- 所属话题分类
- 难度等级（L1/L2/L3）

## 4. 功能模块

### 模块一：每日练习（核心功能）

**定位**：每天 15-20 分钟的集中拼写训练。

#### 模式 A：听音拼写（Spelling Dictation）

这是最核心的练习模式，直接训练「听→拼」的能力。

- 系统朗读一个单词（可重复播放，最多 3 次）
- 同时显示中文释义作为提示
- 学生在输入框中输入英文拼写
- 提交后即时反馈：正确/错误，错误时高亮显示拼错的位置
- 错误单词自动加入「错词本」
- 每轮练习 10-15 个单词

#### 模式 B：释义匹配（Meaning Match）

训练「看到单词 → 反应含义」的速度，辅助阅读理解。

- 显示一个英文单词，提供 4 个中文释义选项
- 限时作答（如 8 秒），超时算错
- 正确/错误即时反馈

#### 模式 C：选词填空（Gap Fill）

在语境中巩固词汇用法。

- 显示一个带空格的英文句子
- 提供 4 个单词选项
- 选择正确单词填入
- 句子来自 PET 真题或模拟题中的典型语境

### 模块二：错词本

**定位**：针对薄弱环节的强化复习。

- 自动收集所有练习中的错误单词
- 按错误次数排序，高频错词优先展示
- 支持「错词专项练习」——只练错词本中的单词
- 连续 3 次拼写正确后，单词自动移出错词本
- 显示每个错词的错误记录（如「把 'accommodation' 拼成 'accomodation'」）

### 模块三：智能复习

**定位**：基于间隔重复算法的长效记忆巩固。

- 采用简化版 SM-2 间隔重复算法
- 练习过的单词按熟练度自动安排复习时间表
- 每日推送「今日待复习」列表（通常 15-30 个词）
- 复习时采用快速闪卡模式：看中文 → 默拼英文 → 翻卡核对
- 标记「记住了」或「还需练习」，影响下次复习间隔

### 模块四：学习进度

**定位**：让家长和学生都能清晰看到学习成效。

- 已学单词数 / 总单词数
- 熟练掌握单词数（连续 3 次以上正确）
- 本周练习天数
- 近 7 天正确率趋势图（简单折线图）
- 每个话题分类的掌握率
- 每日练习时长统计

### 模块五：词汇浏览器

**定位**：按需查阅和自主浏览 PET 词汇库。

- 按话题分类浏览
- 按难度等级筛选（L1/L2/L3）
- 按字母排序浏览
- 关键词搜索（中英文均可）
- 点击单词可查看详情（发音、释义、例句）
- 可手动将单词加入「我的学习计划」

## 5. 用户体验设计

### 5.1 页面结构

```
导航栏（顶部）
├── 今日练习（默认首页）
├── 错词本
├── 智能复习
├── 学习进度
└── 词汇库

内容区（主体）
└── 当前模块的具体内容

底部状态栏（移动端适配）
└── 今日练习进度 | 待复习数量 | 连续打卡天数
```

### 5.2 视觉风格

- **色调**：柔和、护眼的暖色调（浅蓝、浅绿、米白），避免刺眼
- **字体**：清晰易读，英文使用适合拼写的等宽或半等宽字体
- **动画**：简洁克制，仅用于正确/错误反馈（如绿色闪现、红色抖动），无炫目特效
- **图标**：使用清晰的线性图标，辅助理解而非装饰

### 5.3 响应式设计

- 桌面端（≥768px）：左侧导航 + 右侧内容区的双栏布局
- 移动端（<768px）：顶部标签栏切换 + 全宽内容区
- 输入框针对移动端优化：自动聚焦、大字体、键盘类型适配

### 5.4 交互细节

- 单词发音支持 3 种语速（慢速/正常/快速）
- 允许切换英式发音和美式发音
- 练习中途退出时自动保存进度，下次进入可继续
- 所有数据存储在浏览器本地（localStorage），无需登录注册
- 支持数据导出/导入（JSON 格式），方便换设备迁移

## 6. 技术方案

### 6.1 技术选型

| 层面 | 选择 | 理由 |
|------|------|------|
| 框架 | 纯 HTML/CSS/JS 或 React | 单页面应用，无需后端 |
| 发音 | Web Speech API (TTS) | 浏览器原生支持，无需额外音频文件 |
| 数据存储 | localStorage + IndexedDB | 离线可用，数据量较大时用 IndexedDB |
| 部署 | 静态文件托管 | 无需服务器，一个 HTML 文件即可运行 |
| 图标 | SVG 内联 | 减少外部依赖 |

### 6.2 浏览器兼容性

- Chrome 90+
- Safari 14+
- Edge 90+
- 移动端 Chrome 和 Safari

### 6.3 数据安全

- 所有数据保存在用户浏览器本地
- 不涉及任何用户个人信息收集
- 导出功能为纯本地操作

## 8. 开发阶段规划

### Phase 1：MVP（最小可用版本）

- 基础 UI 框架（响应式布局）
- 词汇数据（L1 基础必备，约 300 词先行）
- 核心功能：听音拼写模式（模式 A）
- 错词本基础版
- 学习进度基础版（已学/正确率）
- 数据本地持久化

**目标**：可用的听音拼写练习工具，覆盖电脑和手机。

### Phase 2：功能完善

- 补充释义匹配模式（模式 B）和选词填空模式（模式 C）
- 错词本增强（错误记录、自动移除）
- 智能复习（间隔重复算法）
- 学习进度增强（趋势图、话题掌握率）
- 词汇浏览器
- 词汇数据扩充至 L1 完整 + L2

### Phase 3：体验打磨

- 多语速发音切换
- 英音/美音切换
- 练习进度保存与恢复
- 数据导出/导入
- 词汇数据扩充至 L2 完整 + L3

## 9. 成功指标

### 使用指标（上线后追踪）

- 每周练习天数 ≥ 4 天
- 每次练习完成率 ≥ 80%
- 错词本中单词的 7 天移除率 ≥ 60%（即错词被有效纠正）

### 学习指标（定性观察）

- 学生拼写错误类型从「完全不会拼」转向「偶有字母遗漏」（说明进步）
- 听力练习时能更快反应出单词的拼写形式
- 阅读速度因词汇量增加而提升

## 10. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| Web Speech API 发音不准 | 学生学习错误的发音 | 提供音标辅助；预留预录音频替换方案；允许切换英音/美音 |
| localStorage 数据丢失 | 学习进度丢失 | 提供数据导出功能，鼓励定期备份 |
| 移动端键盘体验差 | 练习效率低 | 拼写输入框使用大字体、自动聚焦、关闭自动纠错 |
| 词汇内容错误 | 学习错误内容 | 词汇数据需人工审核，支持用户反馈纠错 |
| 长期使用动力不足 | 放弃使用 | 通过间隔复习降低认知负担，通过进度可视化增强成就感 |
